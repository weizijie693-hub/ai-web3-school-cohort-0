# Web3 基础概念

> 日期：2026-05-20
> 来源：AI×Web3 School — 自整理笔记

## 账户（Account）

账户是链上状态的基本单位，分为外部账户（EOA，由私钥控制）和合约账户（由代码控制）。

- EOA 由一对公私钥生成，只要有私钥就能在全球任何节点上控制它
- 合约账户部署后就受其代码逻辑控制，不受任何单一私钥支配
- **例子**：你的 MetaMask 里每个网络显示的余额，就是一个 EOA 在各个链上的状态
- **注意**：账户地址不是"账号"——没有找回密码、没有客服。控制私钥就是控制账户

## 地址（Address）

地址是账户在链上的唯一标识符，通常由公钥经哈希运算截取得到。

- Ethereum 地址是 `0x` 开头的 42 位十六进制字符串（如 `0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045`）
- **例子**：给别人转账时，粘贴的那个 `0x...` 字符串就是地址；贴错一个字符，资金就转到无人能控制的地址
- **注意**：不要只复制前几位和后几位来核对——用 Checksum（大小写混合）或 ENS 域名来验证完整地址

## 钱包（Wallet）

钱包是管理私钥并与链交互的软件工具，它并不"存钱"，而是保管访问资金的密钥。

- 钱包的功能：生成密钥、签名交易、连接 DApp、查看余额
- **例子**：MetaMask、Rabby、WalletConnect 都是钱包。切换网络时，同一个地址在不同链上显示不同余额
- **注意**：浏览器扩展钱包的操作本质是"注入一个签名环境"，DApp 无法直接拿走你的私钥，但可以诱骗你签名恶意交易

## 助记词（Seed Phrase）

助记词是 12/24 个英文单词，是私钥的人类可读备份形式，由 BIP-39 标准定义。

- 一组助记词通过分层推导（BIP-32/BIP-44）可以派生出无数个私钥和地址
- **例子**：MetaMask 创建钱包时让你抄下来的那 12 个单词。丢了它 = 丢了所有派生地址的资金
- **安全警告（极高风险）**：
  - **永远不要**把助记词输入任何网站、DApp、表单——包括"验证钱包"、"同步资产"等话术
  - **永远不要**通过网络传输（微信、邮件、截图）
  - **永远不要**存储在联网设备上（云笔记、浏览器收藏夹）
  - 助记词是私钥的根，泄漏它等于交出所有派生账户的完全控制权

## 私钥（Private Key）

私钥是一个 256 位的随机数，是链上所有权的最终凭证——谁持有私钥，谁就拥有对应地址的资产。

- 私钥的数学基础：从私钥可以单向导出公钥和地址，但反过来不可行（椭圆曲线密码学）
- **例子**：在你签名一笔交易时，钱包用私钥生成一个只有你才能产生的签名，网络验证这个签名就知道交易是你授权的
- **安全警告（极高风险）**：
  - 私钥不应以纯文本形式出现在任何日志、报错信息、URL、截图或代码仓库中
  - 开发中如果要用私钥签名（如脚本发交易），使用环境变量或 `.env` 文件，切忌硬编码
  - 一个地址的私钥泄漏，该地址上所有资产（包括跨链桥的资产）即时归攻击者所有，**无法撤销**

## 签名（Signature）

签名是用私钥对一段消息（如交易数据或登录消息）生成的数学证明，让网络验证出"这个请求确实来自私钥持有者且未被篡改"。

- 签名不是加密，不隐藏消息内容——只证明"谁发了什么"
- **例子**：
  - 交易签名：钱包让你确认一笔转账时点击"确认"，背后就是用私钥对交易数据签名
  - 消息签名（Siwe/Sign-in with Ethereum）：用钱包签名一段文本登录网站，网站验证后让你免密登录
- **安全警告（极高风险）**：
  - 签名授权可以轻易包装恶意内容：一个看起来是"登录"的弹窗，实际可能签名了一笔"将你所有 USDC 转给攻击者"的交易
  - **不要盲目签名任何弹窗**——确认你签名的到底是什么（EIP-712 结构化数据让你可以读得懂，但仍需警惕）
  - 钓鱼攻击的核心手法就是伪造签名请求，让用户签一个"批准"操作

## 交易（Transaction）

交易是改变链上状态的唯一方式——任何写操作（转账、部署合约、调用合约）都必须通过交易。

- 交易包含：from、to、value、data、nonce、gas limit、max fee。由发送方签名后广播到网络
- **例子**：你在 MetaMask 里输入接收地址和金额，点击"确认"，钱包构造一个交易并用私钥签名，然后提交到 RPC 节点等待上链
- **注意**：交易一旦被区块确认，**无法撤回**。发送前务必检查：接收地址、金额、网络（链 ID）、Gas 设置

## Gas（燃料费）

Gas 是执行交易或智能合约操作所需的手续费，用以补偿验证者的计算资源消耗。

- Gas = Gas Used × Gas Price。复杂操作（如合约交互）比简单转账消耗更多 Gas
- **例子**：发送 ETH 约消耗 21,000 Gas；在 Uniswap 上做一笔 Swap 可能消耗 150,000+ Gas，因为涉及多次合约调用
- **注意**：Gas 费不是给"矿工"的小费，而是网络安全模型的一部分。Gas 过低会导致交易 stuck（卡住）或被丢弃。网络拥堵时 Gas 价格飙升

## 智能合约（Smart Contract）

智能合约是部署在区块链上的可执行程序，一旦部署就无法修改，运行结果由全网节点独立验证。

- 合约代码和状态公开可见，任何人都可以调用其公开方法（除非有权限控制）
- **例子**：一个 ERC-20 代币合约有一个 `transfer` 方法，你调用它时合约检查你的余额是否足够，然后更新账本状态
- **注意**：智能合约的代码不一定安全。即使经过审计的合约也可能有漏洞（如重入攻击、闪电贷攻击）。与合约交互前应至少了解其功能而非盲目授权

## 测试网（Testnet）

测试网是模拟主网环境的独立区块链，用于开发和测试，其代币没有真实经济价值。

- 常见测试网：Sepolia（Ethereum）、Base Sepolia、Polygon Amur
- **例子**：在 Sepolia 上部署合约、发送测试交易，通过水龙头（Faucet）获取免费测试 ETH
- **注意**：测试网上的"转账成功"不能直接等价于主网同样成功——Gas 机制、MEV 行为、合约交互的竞争条件在主网上完全不同

## 区块浏览器（Block Explorer）

区块浏览器是一个 Web 应用，让你可视化查询链上数据：交易、地址、区块、合约、事件日志。

- **例子**：Etherscan（主网）、Sepolia Etherscan（测试网）。输入交易哈希就能看到状态（Success/Fail）、Gas 消耗、区块号、时间戳等
- **注意**：区块浏览器上显示的"Approval"或"Transaction"不是你实际的操作——它是链上发生了什么。恶意合约可以在你的 UI 显示"成功"但实际做了别的事，浏览器是验证真相的工具

## EOA / 智能账户 / 多签

EOA（Externally Owned Account）由单个私钥控制；智能账户由合约逻辑控制；多签（Multisig）要求多个签名才能执行操作。

| 类型 | 控制方式 | 代表 |
|------|----------|------|
| EOA | 一对私钥 | MetaMask 默认创建的就是 EOA |
| 智能账户 | 合约逻辑 + 自定义规则（如社交恢复、限额） | Safe (formerly Gnosis Safe) |
| 多签 | N 个持有者中 M 个签名通过 | Safe 多签钱包: 3/5 签名才可转账 |

- **例子**：DAO 金库通常用多签管理——任何转账需要团队中多数人签名才生效，避免单点风险
- **注意**：EOA 最简单但最脆弱（私钥丢了什么都没了）。智能账户更灵活但交互成本更高（部署费用、Gas 开销）。多签增加了安全性但也增加了操作延迟

## 需要谨慎处理的四个核心要素

### 1. 私钥 — 最终所有权凭证

私钥是链上资产的**终极控制权**。与 Web2 密码不同：
- 密码可以重置，私钥**无法重置**（没有"忘记密码"功能）
- 密码泄漏可以改，私钥泄漏意味着攻击者获得了与你**完全相同**的控制权
- 运维影响：在脚本、后端或 CI/CD 中使用私钥时，一旦日志打印或 .env 泄漏，资金即时丢失

### 2. 助记词 — 私钥的根

助记词比单个私钥更危险，因为它可以推导出**所有派生地址的私钥**：
- 一个助记词 => 无数个私钥/地址 => 所有派生地址上的所有资产
- 助记词被拍照、截屏或被输入到钓鱼网站 = 所有资产同时丢失
- 安全做法：永远离线保存（物理写下来放保险柜），只在硬件钱包或可信软件钱包首次导入时使用

### 3. 签名 — 授权行为不可逆

签名滥用是 Web3 最常见攻击面：
- **Permit/Approve 签名**：签一次字授权 DApp 使用你的代币，如果授权额度过大（如 `uint256.max`），DApp 可以一次性转走你该代币的全部余额
- **盲签（Blind Signing）**：硬件钱包弹窗显示一堆"看不懂的十六进制"——你签了它，但不一定知道自己授权了什么
- 核心原则：**不明白签的是什么，就不要点确认**

### 4. 交易授权 — 一经确认无法撤回

与 Web2 的数据库事务可以回滚不同：
- 链上交易一旦被打包进区块并被多数节点确认，就是**最终态**
- 反例：发送 ETH 到错误地址，没有任何银行或客服能帮你追回
- 正确定位：每次点击"确认"之前，假设这笔交易**不可逆**，核对接收方、金额、网络、Gas 后再签名

### 总结

| 要素 | 泄漏/误用后果 | 防护原则 |
|------|--------------|---------|
| 私钥 | 该地址所有资产被转走 | 不存网络、不打印日志、不硬编码 |
| 助记词 | 所有派生地址全部失守 | 离线保存、绝不输入任何网站 |
| 签名 | 资产在不发起交易的情况下被转移 | 阅读签名内容、警惕 Approve 授权 |
| 交易确认 | 资金/状态变更不可撤销 | 确认前逐项核对，默认不可逆 |
