# AI × Web3 Concept Explainer

一个交互式概念学习工具，帮助快速理解 AI × Web3 的 11 个核心概念。提供 **CLI 版** 和 **网页版** 两种方式。

- **网页版 (GitHub Pages)**: https://weizijie693-hub.github.io/ai-web3-school-cohort-0/experiments/cli-concept-explainer/index.html
- **CLI 版**: `python cli_concept_explainer.py` (需要 Python 3.6+)

---

## 1. 它解决什么学习问题

AI × Web3 School 的 Handbook 内容丰富，但初学者刚接触时容易遇到三个问题：

| 问题 | 这个工具的解法 |
|------|-------------|
| **概念太多，不知从哪开始** | 把 11 个核心概念集中展示，一键随机学习 |
| **解释太抽象，看了记不住** | 每个概念都用"一句话解释 + 具体例子 + 常见误解"三段式结构呈现，从具体场景切入 |
| **看过就忘，没有检验手段** | 内置 5 题互动测验，通过选择题巩固理解 |

适用场景：碎片时间学习、复习巩固、小组讨论时的快速查词。

---

## 2. 用户如何与它交互

**网页版**（直接打开链接，无需安装）：

| 功能 | 操作 |
|------|------|
| 概念列表 | 打开首页 → 点击任意概念卡片 |
| 查询概念 | 切换 "查询概念" 标签 → 输入关键词 → 按回车或点搜索 |
| 随机学习 | 切换 "随机学习" 标签 → 点按钮 |
| 小测验 | 切换 "小测验" 标签 → 开始答题 → 每题两个选项 → 自动下一题 |

**CLI 版**（终端运行）：

```bash
python cli_concept_explainer.py list              # 列出概念
python cli_concept_explainer.py explain agent      # 查询概念
python cli_concept_explainer.py random             # 随机概念
python cli_concept_explainer.py quiz               # 互动测验
```

## 3. 输入示例和输出示例

### 示例 1：查询概念

**用户输入**：`explain agent`

**输出**：

```
== 智能体 / Agent (AI Agent)
--------------------------------------------------

>> 一句话解释
   智能体是一个能够自主感知环境、做出决策并采取行动来达成目标的 AI 系统，
   而不仅仅是被动地回答单个问题。

>> 具体例子
   一个代码审查 Agent 可以自动 checkout PR 分支、运行静态分析、调用 LLM
   逐文件审查代码变更、在 PR 上留下逐条评论，并在发现严重问题时重新触发
   CI——它全程自主决策，不需要人类逐一下达指令。

>> 常见误解
   初学者常把"调 API 的脚本"和 Agent 混为一谈。Agent 的核心是具备自主
   推理和规划能力：它能在未知情况下降级处理、根据中间结果灵活调整策略。
   单纯的条件分支编排不属于 Agent，只是自动化脚本。
```

## 4. 哪部分由 AI 生成，哪部分由你人工修改 / 验证

| 部分 | AI 生成 | 人工验证 / 修改 |
|------|---------|----------------|
| **概念内容**（11 个概念的三种解释） | AI 生成初稿，基于模型自身知识和 Handbook 内容 | 逐条审校，修正表述不准确之处；为"常见误解"补充真实案例场景 |
| **三段式结构**（一句话/例子/误解） | AI 建议该格式 | 人工确认该结构适合学习场景，决定最终模板 |
| **概念数据文件** (`concepts.json`) | JSON 结构和内容初稿由 AI 生成 | 逐条核对 Handbook 一致性，修正区块链/PoS 等技术概念的精确性 |
| **CLI 工具代码** (`cli_concept_explainer.py`) | AI 生成基础框架，包含 list/explain/random/quiz 命令 | 人工调试 Windows GBK 编码兼容性（替换 emoji，确保中文输出正常）；修复空输入、边界条件 |
| **网页版** (`index.html`) | AI 生成 HTML/CSS/JS 结构和交互逻辑 | 人工调整 UI 布局、颜色和响应式设计；添加 AI-assisted / Human-verified 标签系统 |
| **测验逻辑** | AI 设计基础方案 | 人工添加干扰项合理性过滤、空输入容错、分数统计 |
| **README** | AI 协助撰写 | 人工补充 AI/Human 分工的部分，确保说明真实准确 |

### 人工审核的具体案例

**案例 1 — 概念修正**：AI 生成的 LLM 解释初稿中，将"幻觉"归因于训练数据不足。人工审校后补充了更准确的原因：LLM 本质是基于统计规律生成下文而非事实检索。

**案例 2 — 编码兼容性**：AI 生成的 CLI 代码使用了 emoji 字符（📖🔹🔸⚠️），在 Windows 终端（GBK 编码）中 crash。人工调试后用 ASCII 等价替换，确保跨平台可用。

**案例 3 — 测验容错**：AI 设计的测验没有处理空输入和无效输入的情况。人工补充了边界条件处理逻辑。

## 5. 还有哪些限制或下一步改进

### 当前限制

- **概念数量有限**：目前只有 11 个核心概念，Handbook 还有大量内容未覆盖
- **无用户进度追踪**：测验分数和查看记录不会保存，刷新后丢失
- **单用户**：不支持多用户或协作学习
- **无动态生成**：所有概念内容是静态的，不能根据用户输入动态生成新解释

### 下一步改进方向

| 方向 | 描述 | 难度 |
|------|------|------|
| **扩充概念库** | 覆盖 Handbook 全部章节，增加到 30+ 概念 | 低 |
| **社区贡献机制** | 开放概念内容的 PR，增加贡献者指引 | 中 |
| **进度持久化** | 使用 localStorage 保存学习进度和测验记录 | 低 |
| **暗色模式** | 增加主题切换 | 低 |
| **英文版** | 双语切换功能 | 中 |
| **AI 动态生成** | 接入 LLM API，允许用户对概念追问或生成个性化例子 | 高（需要 API Key 管理） |
| **间隔复习** | 根据遗忘曲线安排概念复习提醒 | 中 |
